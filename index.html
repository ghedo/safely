NAME

safely - pretty command-line password manager

SYNOPSIS

safely [OPTIONS] COMMAND [ACCOUNT]

DESCRIPTION

Safely is a command-line tool to manage passwords. It stores all your login information (usernames and passwords) in a simple, JSON-formatted file, encrypted and signed using your OpenPGP key: this makes it pretty secure and allows manual decryption of the database when needed.

COMMANDS

-c, --create

       Create a new password database.

-a, --add

       Add the given account.

-p, --passwd

       Show the given account's password.

-u, --user

       Show the given account's username.

-e, --edit

       Edit the given account.

-r, --remove

       Remove the given account.

-s, --search

       Search for the given pattern.

-d, --dump

       Dump JSON database.

-t, --tests

       Run the security tests only.

-v, --version

       Show program's version and exit.

-h, --help

       Show program's help and exit.

OPTIONS

-D, --db

       Specifies a custom path to the password database.

-K, --keys

       Space-separated list of keys to be used.

-H, --hide

       Print the passwords in red on red to avoid shoulder surfing.

-Q, --quiet

       Output errors only (i.e. no "[ok]" messages).

-F, --fuzzy

       Search for non-exact (fuzzy) matches.

-S, --nosecure

       Ignore any security test failure.

-B, --nobackup

       Do not create database file backups.

-A, --noagent

       Do not use the default gpg agent.

ENVIRONMENT

GPG_AGENT_INFO

       Used to locate the gpg-agent. This is only used when --noagent is not set.

GPG CONFIGURATION

Before starting, you may want to add the following config to your gpg.conf:

default-key <MYKEY>
default-recipient-self

Where <MYKEY> is the ID of your key. Otherwise the --keys option must be provided.

FILES

~/.safely.db

       The default password database file. The option --db can be used to override it.

~/.safely.db~

       The default password database backup file. If the --db option is set, this will change accordingly. If the --nobackup option is used, this file is not created/modified.

~/.safely.db.lock

       The default password database lock file. If the --db option is set, this will change accordingly.
